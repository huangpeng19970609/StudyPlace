##### 1  认识闭包

  1. `外部的作用域链`虽然在函数执行完毕后`销毁`了， 但由于`内部函数的作用域链`依旧被引用（被引用了，没被销毁）的原因， 所以外部函数的活动对象也`依旧保存`在内存中！
- 活动对象 只是一个引用的地址
- 实际使用中， 我们总是喜欢用一个对象包裹这些函数，再返回。

```js

 		function fn { color: 'red'};  例: fn的活动对象 => 全局活动对象
 =>

 但【闭包情况下】，
```
